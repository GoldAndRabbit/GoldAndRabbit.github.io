2.еҜ№дәҺеҸҳйҮҸ, жҲ‘д»¬йҖҡеёёдёҚйңҖиҰҒзҹҘйҒ“жҜҸдёӘеҸҳйҮҸиў«е…·дҪ“еҲҶй…ҚеҲ°дәҶеҶ…еӯҳзҡ„ең°еқҖжҳҜд»Җд№Ҳ, з”ЁдәҶеӨҡе°‘дёӘbytesеҺ»еӯҳеӯҳеӮЁе®ғ; жҲ‘д»¬еҸӘжҳҜйҖҡиҝҮе®ғзҡ„еҗҚеӯ—еңЁдҪҝз”Ёе®ғ, е№¶дё”жҲ‘д»¬зҹҘйҒ“вҖқе®ғжҳҜжңүдёӘзЎ®е®ҡзҡ„ең°еқҖвҖқзҡ„, жң¬иҙЁдёҠжҳҜзј–иҜ‘еҷЁе®ҢжҲҗдәҶд»ҺвҖқеҸҳйҮҸзҡ„ж ҮиҜҶз¬ҰвҖқеҲ°вҖқеҶ…еӯҳзҡ„е…·дҪ“ең°еқҖвҖқд№Ӣй—ҙзҡ„вҖқзҝ»иҜ‘вҖқ, иҝҷдёӘжҳ е°„ең°еқҖзҡ„иҝҮзЁӢ (addressing, еҜ»еқҖ), е®Ңе…Ёиў«зј–иҜ‘еҷЁжүҳз®Ў

The dereference operator (*) и§Јеј•з”Ёж“ҚдҪңз¬Ұ

1.еҜ№дёҖдёӘеҸҳйҮҸжқҘиҜҙ, иҺ·еҸ–еҸҳйҮҸзҡ„ең°еқҖжң¬иә«жІЎжңүд»Җд№Ҳз”ЁеӨ„, е…ідәҺең°еқҖдёҠжҲ‘д»¬иғҪеҒҡзҡ„жңҖжңүз”ЁдәӢиҝҳжҳҜиҝҳжҳҜиҺ·еҸ–еӯҳеӮЁеңЁең°еқҖдёҠзҡ„еҸҳйҮҸ; жҲ‘д»¬еҸҜд»ҘжңүдәҶеҸҳйҮҸзҡ„ең°еқҖзҡ„ж—¶еҖҷ, з”Ё dereference operator (и§Јеј•з”Ёиҝҗз®—з¬Ұ) еӯҳеӮЁеңЁжҹҗдёӘең°еқҖдёҠзҡ„еҖј
2.еҸ–ең°еқҖиҝҗз®—з¬Ұ вҖң&вҖқ е’Ң и§Јеј•з”Ёиҝҗз®—з¬ҰвҖқ*вҖқ иҝҷдёӨдёӘе…ідәҺең°еқҖзҡ„иҝҗз®—з¬Ұ, зұ»дјјдёҖз§ҚйҖҶиҝҗз®—дёҖж ·, еҜ№дәҺдёҖдёӘеҸҳйҮҸжқҘиҜҙ, дёҖдёӘеҸҜд»ҘжӢҝеҲ°xзҡ„ең°еқҖ, дёҖдёӘеҸҜд»ҘжӢҝеҲ°xең°еқҖдёӢзҡ„еҖј 

#include <iostream>
int main() {
  int x{5};
  std::cout << x << '\n';     // print the value of variable x
  std::cout << &x << '\n';    // print the memory address of variable x
  std::cout << *(&x) << '\n'; // print the value at the memory address of variable x (parentheses not required, but make it easier to read)
  return 0;
}

Pointers жҢҮй’Ҳ

A pointer is an object that holds a memory address (typically of another variable) as its value. This allows us to store the address of some other object to use later.

In modern C++, the pointers we are talking about here are sometimes called вҖңraw pointersвҖқ or вҖңdumb pointersвҖқ, to help differentiate them from вҖңsmart pointersвҖқ.

intuitively,
1.Pointer (жҢҮй’Ҳ) жҳҜдёҖдёӘ object, иҝҷдёӘ object зҡ„еҖјдҝқеӯҳзҡ„жҳҜдёҖдёӘе…¶д»–еҸҳйҮҸзҡ„еҶ…еӯҳең°еқҖ; иҝҷйҮҢеҸҜд»ҘеҜ№жҜ”дёӢ reference еј•з”Ё, еј•з”ЁдёҚжҳҜдёҖдёӘ object
2.зҺ°д»Ј cpp йҮҢйқў, жҲ‘д»¬йҖҡеёёи°ҲеҲ°зҡ„жҢҮй’ҲйҖҡеёёеҸ«еҒҡ вҖңraw pointersвҖқ жҲ–иҖ… вҖңdumb pointersвҖқ , дё»иҰҒжҳҜеҢәеҲ«дәҺжҷәиғҪжҢҮй’Ҳ вҖңsmart pointersвҖқ
3.жҢҮй’Ҳзұ»еһӢзҡ„еЈ°жҳҺе’ҢжҢҮй’Ҳзҡ„dereference, з”Ёзҡ„жҳҜзӣёеҗҢзҡ„з¬ҰеҸ· *, жҜ”еҰӮint *; иҝҷйҮҢе…¶е®һжҳҜдёәд»Җд№ҲжҢҮй’Ҳжңүж—¶еҖҷи®©дәәиҝ·жғ‘зҡ„з¬¬дёҖдёӘеҺҹеӣ , жңүж—¶еҖҷ * дҪңдёәдёҖз§Қ dereference дҪңз”ЁеҺ»еЈ°жҳҺ, жңүж—¶еҖҷдҪңдёәдёҖз§Қзұ»еһӢе®ҡд№үзҡ„з¬ҰеҸ·еҮәзҺ°, жүҖд»ҘеңЁзҗҶи§ЈиҝҷдёӘжҰӮеҝөзҡ„ж—¶еҖҷ, иҰҒжҳҺзЎ®еҲҶжё…жҘҡд»ЈиЎЁзҡ„дёҚеҗҢзҡ„е…·дҪ“зҡ„еҗ«д№ү
4.еңЁеҗҢдёҖиЎҢд»Јз ҒдёӯеЈ°жҳҺ2дёӘжҢҮй’Ҳзұ»еһӢзҡ„еҸҳйҮҸзҡ„ж—¶еҖҷ, йңҖиҰҒе°Ҷзұ»еһӢе®ҡд№үеҝ…йЎ»еҲҶејҖ, еҗҰеҲҷз¬¬дәҢдёӘеҸҳйҮҸдјҡеЈ°жҳҺжҲҗintзұ»еһӢ; жҲ–иҖ…еҸӘиҰҒе®ҡд№үжҢҮй’Ҳ, е°ұжҜҸдёӘжҢҮй’ҲеҸҳйҮҸеҚ•зӢ¬еҶҷдёҖиЎҢйҒҝе…Қе…¶д»–зҡ„й—®йўҳ 

int;                // a normal int
int&;               // an lvalue reference to an int value
int*;               // a pointer to an int value (holds the address of an integer value)
int* ptr1, ptr2;    // incorrect: ptr1 is a pointer to an int, but ptr2 is just a plain int!
int* ptr3, * ptr4;  // correct: ptr3 and ptr4 are both pointers to an int

Pointer initialization жҢҮй’ҲеҲқе§ӢеҢ–

intuitively,
1.зӣёжҜ”дёҖиҲ¬зҡ„еҸҳйҮҸ (жҲ–иҖ…еј•з”Ё) , жҢҮй’Ҳзҡ„еҲқе§ӢеҢ–йңҖиҰҒйқһеёёе°Ҹеҝғең°еҺ»зҗҶи§Је’ҢдҪҝз”Ё
2.е’ҢдёҖиҲ¬еҸҳйҮҸзұ»дјј, жҢҮй’ҲеҸҜд»Ҙй»ҳи®ӨдёҚиў«еҲқе§ӢеҢ–зҡ„, дҪҶжҳҜдёҚеҲқе§ӢеҢ–жҢҮй’ҲеҸҜиғҪдјҡеёҰжқҘзӣёеә”зҡ„йЈҺйҷ©
3.дёҖдёӘжІЎжңүиў«еҲқе§ӢеҢ–зҡ„жҢҮй’ҲеҸ«еҒҡ wild pointer (йҮҺжҢҮй’Ҳ), wild pointer еҢ…еҗ«дёҖдёӘ garbage address (еһғеңҫең°еқҖ), еҜ№дёҖдёӘ wild pointer иҝӣиЎҢ dereference ж“ҚдҪңдјҡеҜјиҮҙ undefined behavior (жңӘе®ҡд№үзҡ„иЎҢдёә)
4.еӣ дёә wild pointer зҡ„ж“ҚдҪңеҸҜиғҪеј•иө·жңӘе®ҡд№үзҡ„иЎҢдёә, жүҖд»ҘжҲ‘д»¬еңЁдҪҝз”Ёзҡ„ж—¶еҖҷ always initialize pointers. дёҚз®Ўзј–иҜ‘еҷЁе…ҒдёҚе…Ғи®ёе®ҡд№үйҮҺжҢҮй’Ҳ, жҲ‘д»¬еҝ…йЎ»еҲқе§ӢеҢ–жҢҮй’Ҳ
5.жҢҮй’Ҳдёәд»Җд№ҲеҸ«вҖқжҢҮй’ҲвҖқ? жҜ”еҰӮжҢҮй’ҲеҸҳйҮҸ ptr дҝқеӯҳзҡ„жҳҜеҸҳйҮҸ x зҡ„ең°еқҖ, жҲ‘д»¬иҜҙ ptr вҖңжҢҮеҗ‘дәҶвҖқ x
6.жҲ‘д»¬йҖҡеёёдёҖз§ҚиҜҙжі•еҸ«еҒҡ вҖңintжҢҮй’ҲвҖқ вҖңcharжҢҮй’ҲвҖқ зӯү вҖңxxжҢҮй’ҲвҖқ е…¶дёӯ xx жҳҜдёҖз§Қзұ»еһӢ, иҝҷз§ҚиҜҙжі•жҳҜ вҖңpointer to an xвҖқ зҡ„дёҖз§Қзј©еҶҷ 

int main() {
  int x{5};
  int* ptr;                   // еЈ°жҳҺдёҖдёӘжңӘеҲқе§ӢеҢ–зҡ„жҢҮй’Ҳ, дҪҶжҳҜдёҚжҢҮеҗ‘д»»дҪ•еҸҳйҮҸ
  int* ptr2{};                // дёҖдёӘnull pointer
  int* ptr3{&x};              // жҢҮеҗ‘xзҡ„жҢҮй’Ҳ, з”Ёxзҡ„ең°еқҖиҝӣиЎҢжҢҮй’Ҳзҡ„еҲқе§ӢеҢ–
  std::cout << *ptr3 << '\n'; // з”Ёdeferenceз¬ҰеҸ·еҺ»иҺ·еҸ–ptrжҢҮеҗ‘зҡ„еҖј
  return 0;
}

7.жҢҮй’Ҳе’Ңеј•з”ЁжңүдёӘзӣёдјјзӮ№: жҢҮй’ҲжҢҮеҗ‘зҡ„еҸҳйҮҸзұ»еһӢе’ҢжҢҮй’Ҳзҡ„зұ»еһӢеҝ…йЎ»иҰҒеҢ№й…ҚдёҠ (еҗҢзҗҶеј•з”ЁеҸҳйҮҸзұ»еһӢе’Ңиў«еј•з”Ёзҡ„зұ»еһӢ) 

int main() {
  int i{5};
  double d{7.0};
  int* iPtr{&i};      // ok: a pointer to an int can point to an int object
  int* iPtr2{&d};     // not okay: a pointer to an int can't point to a double object
  double* dPtr{&d};   // ok: a pointer to a double can point to a double object
  double* dPtr2{&i};  // not okay: a pointer to a double can't point to an int object
  return 0;
}

Pointers and assignment жҢҮй’Ҳзӣёе…ізҡ„иөӢеҖјж“ҚдҪң

жңүдәҶжҢҮй’Ҳд№ӢеҗҺ, жҲ‘д»¬зҡ„иөӢеҖјж“ҚдҪңд»ҺжӯӨеӨҡдәҶдёҖз§Қж–№жі•: еҲ©з”ЁжҢҮй’ҲиҝӣиЎҢиөӢеҖј, е…·дҪ“иҖҢиЁҖ, жҲ‘д»¬еҲ©з”ЁжҢҮй’ҲиҝӣиЎҢиөӢеҖјдёӨз§ҚиөӢеҖјзҡ„ж–№жі•
1.ж”№еҸҳзҡ„жҢҮй’ҲжҢҮеҗ‘зҡ„ең°еқҖ, д№ҹе°ұжҳҜз»ҙжҢҮй’ҲеҲҶй…Қж–°ең°еқҖ; иҝҷз§Қжғ…еҶөжҲ‘д»¬зҡ„ж“ҚдҪңжҳҜ 

int x{3};
int y{5};
int* ptr; 
ptr = &x;
ptr = &y;

2.ж”№еҸҳзҡ„жҳҜиў«жҢҮеҗ‘зҡ„еҖј, д№ҹе°ұжҳҜеҲҶй…Қз»ҷжҢҮй’ҲжҢҮеҗ‘зҡ„еҖјдёҖдёӘж–°зҡ„еҖј; иҝҷз§Қжғ…еҶөжҲ‘д»¬йҮҮз”Ёзҡ„жҳҜ dereference иҝҗз®—з¬ҰзӣҙжҺҘиөӢеҖј *ptr = 5 

int x{3};
int* ptr; 
*ptr = &x;
*ptr = 5;

з¬¬ 1 з§Қжғ…еҶөдёӢ, жҲ‘д»¬жҳҜдҝ®ж”№дәҶжҢҮй’Ҳзҡ„ең°еқҖ 

#include <iostream>
int main() {
  int x{5};
  int* ptr{&x};               // ptrдёҖејҖе§ӢжҢҮеҗ‘дәҶx
  std::cout << *ptr << '\n';  // print the value at the address being pointed to (x's address)
  int y{6};
  ptr = &y;                   // ptrйҮҚж–°жҢҮеҗ‘дәҶy 
  std::cout << *ptr << '\n';  // print the value at the address being pointed to (y's address)
  return 0;
}

Pointers v.s. lvalue references жҢҮй’Ҳе’Ңеј•з”Ёзҡ„еҜ№жҜ”

жҢҮй’Ҳе’Ңеј•з”Ёзҡ„зӣёдјјзӮ№:
1.жҢҮй’Ҳе’Ңеј•з”ЁеңЁеҫҲеӨҡжғ…еҶөдёӢдҪҝз”Ёзҡ„ж—¶еҖҷжҳҜйқһеёёзұ»дјјзҡ„, жң¬иҙЁйғҪжҳҜйҖҡиҝҮз»‘е®ҡеҸҳйҮҸе®һзҺ°
2.жҢҮй’Ҳе’Ңеј•з”ЁеңЁеҜ№еҗҢдёҖдёӘеҸҳйҮҸж··з”Ёзҡ„ж—¶еҖҷ, е®һзҺ°зҡ„иөӢеҖјж“ҚдҪңжҳҜзӯүж•Ҳзҡ„, еҸҜд»ҘйҖҡиҝҮдёӢйқўзҡ„caseзҗҶи§Ј

жҢҮй’Ҳе’Ңеј•з”Ёзҡ„дёҚеҗҢзӮ№:
1.еҜ»еқҖе’Ңdefererenceж“ҚдҪңзҡ„жҳҫејҸиҝҳжҳҜйҡҗејҸ, жҢҮй’ҲеңЁдҪҝз”Ёзҡ„ж—¶еҖҷжҲ‘д»¬жҳҫзӨәең° (explicitly) иҺ·еҸ–дәҶжҢҮеҗ‘ең°еқҖ, еҗҢж—¶жҳҫзӨәең° (explicitly) йҖҡиҝҮ dereference жҢҮй’Ҳж“ҚдҪңеҺ»иҺ·еҸ–еҖј; иҖҢеј•з”Ёзҡ„вҖқеҜ»еқҖвҖқе’Ң deference ж“ҚдҪңжҳҜйҡҗејҸ (implicitly) зҡ„
2.еј•з”Ёеҝ…йЎ»иў«еҲқе§ӢеҢ–, жҢҮй’Ҳй»ҳи®ӨдёҚиў«еҲқе§ӢеҢ– (дҪҶжҳҜдҪңдёәdeveloperеә”иҜҘд»ҘеҲқе§ӢеҢ–дҪңдёәеҝ…иҰҒеҺҹеҲҷ)
3.еј•з”ЁдёҚжҳҜ objects, дҪҶжҳҜжҢҮй’ҲжҳҜ objects
4.еј•з”ЁдёҚиғҪдҝ®ж”№, жҢҮй’ҲеҸҜд»Ҙд»»ж„Ҹдҝ®ж”№жҢҮеҗ‘зҡ„еҜ№иұЎ, еӣ жӯӨжҢҮй’ҲжҜ”еј•з”ЁжӣҙеҠ зҒөжҙ», д»ҺеҸҰдёҖдёӘи§’еәҰжқҘиҜҙд№ҹжӣҙе®№жҳ“еҮәй”ҷ
5.еј•з”Ёж°ёиҝңйңҖиҰҒз»‘е®ҡеңЁдёҖдёӘ objects дёҠйқў, дҪҶжҳҜжҢҮй’ҲеҸҜд»ҘжҢҮеҗ‘ nothing
6.еј•з”ЁеңЁдҪҝз”ЁдёҠзҡ„йҖҡеёёе®үе…ЁжҖ§жӣҙејә, жҢҮй’ҲеҫҲзҒөжҙ», дҪҶжҳҜе®№жҳ“еҗ„з§ҚеҮәй—®йўҳ 

#include <iostream>
int main() {
  int x{5};
  int& ref {x};  // refз»‘е®ҡдәҶx
  int* ptr {&x}; // ptrжҢҮеҗ‘дәҶx

  std::cout << x;
  std::cout << ref;          // зӣҙжҺҘдҪҝз”ЁrefжқҘжү“еҚ°еҸҳйҮҸx
  std::cout << *ptr << '\n'; // зӣҙжҺҘдҪҝз”Ё*ptrжү“еҚ°еҸҳйҮҸ;

  ref = 6;                   // use the reference to change the value of x
  std::cout << x;
  std::cout << ref;          // use the reference to print x's value (6)
  std::cout << *ptr << '\n'; // use the pointer to print x's value (6)

  *ptr = 7;                   // use the pointer to change the value of x
  std::cout << x;
  std::cout << ref;           // use the reference to print x's value (7)
  std::cout << *ptr << '\n';  // use the pointer to print x's value (7)
  return 0;
}

The size of pointers жҢҮй’Ҳзҡ„еӨ§е°Ҹ

1.жҢҮй’Ҳзҡ„ size еҸ–еҶідәҺжһ¶жһ„, еҗҢж—¶ж— и§ҶжҢҮеҗ‘зҡ„зұ»еһӢ, дёҚз®ЎжҳҜжҳҜд»Җд№Ҳзұ»еһӢ, еӨ§е°ҸжҖ»жҳҜзӣёеҗҢзҡ„, еӣ дёәеҸӘжҳҜеҶ…еӯҳең°еқҖ, ең°еқҖжүҖйңҖиҰҒдҝқеӯҳзҡ„bitsж•°йҮҸжҳҜдёҖдёӘеӣәе®ҡеҖј 

#include <iostream>
int main() {
  char* chPtr{};        // chars are 1 byte
  int* iPtr{};          // ints are usually 4 bytes
  long double* ldPtr{}; // long doubles are usually 8 or 12 bytes

  std::cout << sizeof(chPtr) << '\n'; // prints 8 
  std::cout << sizeof(iPtr) << '\n';  // prints 8
  std::cout << sizeof(ldPtr) << '\n'; // prints 8
  return 0;
}

Dangling pointers жӮ¬з©әжҢҮй’Ҳ

a dangling pointer is a pointer that is holding the address of an object that is no longer valid (e.g. because it has been destroyd)

Dereferencing a dangling pointer (e.g. in order to print the value being pointed at) will lead to undefined behavior, as you are trying to access an object that is no longer valid.

Perhaps surprisingly, the standard says вҖңAny other use of an invalid pointer value has implementation-defined behaviorвҖқ. This means that you can assign an invalid pointer a new value, such as nullptr (because this doesnвҖҷt use the invalid pointerвҖҷs value). However, any other operations that use the invalid pointerвҖҷs value (such as copying or incrementing an invalid pointer) will yield implementation-defined behavior.

intuitively,
1.dangling pointer (жӮ¬з©әжҢҮй’Ҳ) е°ұжҳҜиҜҙдёҖдёӘжҢҮй’Ҳд»ЈиЎЁзҡ„ең°еқҖдёҠзҡ„ object е·Із»Ҹй”ҖжҜҒдәҶ&ж— ж•ҲдәҶ, жүҖд»ҘиҝҷдёӘжҢҮй’ҲеҘҪжҜ”жӮ¬з©әзҡ„жҢӮиө·жқҘдёҖж ·
2.еҜ№дәҺдёҖдёӘжӮ¬з©әзҡ„жҢҮй’Ҳ, дјҡеҜјиҮҙ undefined behavoir (жңӘе®ҡд№үиЎҢдёә); еҜ№дёҖдёӘ dangling pointer йҮҮз”Ё dereference ж“ҚдҪңе°ұжҳҜжңӘе®ҡд№үиЎҢдёә
3.cpp ж ҮеҮҶиЎЁзӨәвҖқж— ж•ҲжҢҮй’ҲеҖјзҡ„д»»дҪ•е…¶д»–дҪҝз”ЁйғҪе…·жңүе®һзҺ°е®ҡд№үзҡ„иЎҢдёәвҖқ. иҝҷж„Ҹе‘ізқҖеҸҜд»Ҙдёәж— ж•ҲжҢҮй’ҲеҲҶй…ҚдёҖдёӘж–°еҖј, дҫӢеҰӮ nullptr (еӣ дёәиҝҷдёҚдҪҝз”Ёж— ж•ҲжҢҮй’Ҳзҡ„еҖј)гҖӮ дҪҶжҳҜпјҢдҪҝз”Ёж— ж•ҲжҢҮй’ҲеҖјзҡ„д»»дҪ•е…¶д»–ж“ҚдҪңпјҲдҫӢеҰӮеӨҚеҲ¶жҲ–йҖ’еўһж— ж•ҲжҢҮй’Ҳпјүе°Ҷдә§з”ҹе®һзҺ°е®ҡд№үзҡ„иЎҢдёә 

#include <iostream>

int main() {
  int x{5};
  int* ptr{&x};
  std::cout << *ptr << std::endl; // valid
  {
    int y{6};
    ptr = &y;
    std::cout << *ptr << std::endl; // valid
  } // y goes out of scope, and ptr is now dangling

  std::cout << *ptr << std::endl; // undefined behavior from dereferencing a dangling pointer
  return 0;
}
